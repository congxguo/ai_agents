# Skill: Morning Investor Scan

## Purpose
Run a once-daily, pre-market scan of a curated AI/data-platform stock universe to surface
new position-trade candidates (weeks-to-months holding horizon), combining fundamental
quality and technical timing, and deliver the ranked list with reasoning to the user via
Telegram. This is a pure advisory skill — it does not place orders, track approvals, or
take any action on the user's account. The user reviews the advice and executes any
trades manually.

## Trigger
- Scheduled, once per trading day, ~07:30 ET (before market open).
- Should be skippable/idempotent: if already run today, re-running should not re-post
  duplicate candidates (check a daily run-log before posting).

## Inputs
- `universe.yaml` — list of ticker symbols (AI/data-platform domain), maintained by the
  user, not regenerated by the agent.
- moomoo OpenD connection (prices, daily/weekly bars, technicals, account/positions).
- FMP API key (fundamentals: financial statements, ratios, growth metrics).
- Telegram bot credentials (send message to user/chat).
- Prior holdings/portfolio state (via moomoo account query, used for portfolio-fit
  context only — read-only).
- Persistent store (file or lightweight DB) for run history and candidate logs.

## Outputs
- A Telegram message (or set of messages) per session: 0–10 ranked candidates, each with
  symbol, entry zone, suggested size, invalidation/stop level, and a short rationale.
- A log entry per candidate generated this session, written to the persistent store, for
  later review of screen quality over time.

## Loop steps

1. **Check run state**
   - Read today's run-log entry. If a scan already completed today, exit (no-op).
   - If not, proceed and create today's run-log entry.

2. **Load universe**
   - Read `universe.yaml`. Do not auto-modify; if empty or missing, abort and alert
     via Slack ("universe config missing — scan skipped").

3. **Fetch market data (moomoo)**
   - For each symbol: daily/weekly OHLCV (enough history for 50/200-day MAs, recent
     relative strength vs. a sector benchmark), and latest quote.
   - On a per-symbol fetch failure: log and skip that symbol, continue with the rest.
     Do not abort the whole run for one bad symbol.

4. **Fetch fundamentals (FMP, cached)**
   - For each symbol: latest financial ratios, growth metrics, recent earnings.
   - Check cache first (e.g., refresh fundamentals at most once/day per symbol) to stay
     within the free-tier daily call budget.
   - On fetch failure: fall back to cached data if available; otherwise skip symbol
     from this run and log the gap.

5. **Fundamental filter**
   - Apply deterministic rules (config-driven, not hardcoded) to the universe, e.g.:
     positive revenue/earnings growth, valuation within a defined band vs. sector peers,
     balance sheet health thresholds.
   - Output: a "fundamentally qualified" subset.

6. **Technical filter**
   - On the qualified subset, apply timing rules, e.g.: price above a rising long-term
     moving average, pullback-to-support in an uptrend, or relative strength vs. sector
     benchmark above a threshold.
   - Output: a "qualified + well-timed" candidate set.

7. **Portfolio fit check**
   - Pull current holdings/cash from moomoo.
   - Drop or flag candidates that would breach: max position count, sector concentration
     limit, correlation with existing holdings, or available cash for a normal-sized
     position.
   - Output: final candidate set (target size: top 5–10, ranked).

8. **Rationale synthesis (LLM step)**
   - For each final candidate, generate a short rationale combining the fundamental and
     technical signals plus any notable recent news/earnings context.
   - This step explains *why now* — it does not re-decide *whether* the candidate
     qualifies (that's steps 5–7, deterministic).

9. **Post to Telegram**
    - Send one message (or a short batch of messages) listing the final candidates:
      symbol, entry zone, suggested size, invalidation level, rationale.
    - If zero candidates pass all filters, send a short "no candidates today" message
      rather than staying silent (silence should not be mistaken for a missed run).

10. **Log & close run**
    - Write full record of this run: candidates generated, filters passed/failed,
      rationale sent. Mark run-log entry complete.
    - No approval tracking, no order placement — the skill's responsibility ends once
      the advice is delivered.

## Guardrails
- This skill never places orders or interacts with the moomoo trading API beyond
  read-only account/position queries for portfolio-fit context.
- Fundamental and technical filters are deterministic, config-driven rules — the LLM
  step only synthesizes rationale, it does not gate candidate selection.
- A failure in one data source (single symbol or full source outage) degrades the
  candidate set rather than crashing the run; always notify via Telegram if the run
  completed in a degraded state (e.g., "FMP unavailable, fundamentals stale").
- Universe list is user-maintained, not agent-modified.
- One run per day; no intraday re-scanning.

## Out of scope (explicitly not part of this skill)
- Any order placement or trade execution (manual, by the user, based on the advice).
- Approval tracking / interactive decisioning via Telegram (no reaction/reply handling).
- Intraday monitoring.
- Position exit management (stop/target tracking) — separate skill, if built later.
- Universe discovery/expansion — separate skill, if built later.
- Backtesting/threshold tuning — separate offline process, informed by this skill's logs.
